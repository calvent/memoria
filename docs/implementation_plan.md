# 回忆录数据库数据填充实现计划

## 目标

将前端`memoria_desgin`中的mock数据转换为真实的数据库记录,为后端系统创建完整的测试数据。基于真实历史背景(知青、上山下乡、改革开放等)设计符合时代特征的人物和故事。

包括:
- 4个用户(张天明、李秀芳、王建国、刘和平)
- 每个用户的老人档案(设计了完整的人生轨迹)
- 4个回忆录,总计19个章节
- 每个章节包含2-8条详细的回忆内容

## 背景

当前前端原型使用的是`SAMPLE_PUBLIC_MEMOIRS`等硬编码的mock数据。后端数据库已经定义了完整的数据模型,但缺少测试数据。

用户要求:
1. 考虑真实的中国社会和上海历史背景(知青、上山下乡、文革等)
2. 为每个人物设计合理的人生轨迹
3. 任务设计成原子性的,支持断点续传
4. 所有内容写入文档,方便中断后继续

详细人物档案见 [characters.md](file:///Users/liaochui/.gemini/antigravity/brain/461676d9-36d2-4ce9-8742-3867fc43e812/characters.md)

### 原子化任务设计

将内容编写拆分为**19个原子任务**(每个章节一个任务),每个任务包含:
- 明确的时间跨度
- 核心主题
- 2-8个回忆点建议
- 编写状态追踪(⏸️待编写、▶️进行中、✅已完成)

这样设计的好处:
1. **独立性**: 每个章节可以单独完成,不依赖其他章节
2. **可中断**: 网络中断后可以从任何章节继续
3. **可追踪**: 通过状态标记清楚了解进度
4. **灵活性**: 可以并行编写多个章节或调整顺序

详细计划见 [content_plan.md](file:///Users/liaochui/.gemini/antigravity/brain/461676d9-36d2-4ce9-8742-3867fc43e812/content_plan.md)
