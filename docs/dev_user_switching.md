# 开发环境用户切换指南

## 🎯 功能说明

在开发环境下，"我的"页面提供了一个方便的用户切换功能，让你可以快速切换不同的测试账号来查看不同用户的数据。

---

## 📱 使用方法

### 1. 进入"我的"页面

在小程序底部导航栏点击"我的"图标

### 2. 找到用户切换器

在用户信息卡片下方，你会看到一个橙色背景的"开发模式 - 切换测试用户"面板

### 3. 选择要切换的用户

点击用户选择器，会弹出选择列表：

- **张天明** (工厂工人 - 3个章节)
- **李秀芳** (乡村教师 - 5个章节)  
- **王建国** (内科医生 - 7个章节)
- **刘和平** (钢铁工人 - 4个章节)

### 4. 确认切换

选择后会自动：
1. 调用手机号登录API
2. 更新本地token
3. 刷新页面
4. 显示切换成功提示

## ⚙️ 技术实现

### 自动登录逻辑

```typescript
// frontend/src/services/request.ts
const autoLogin = async () => {
  if (IS_DEV) {
    // 开发环境默认使用张天明账号
    const response = await Taro.request({
      url: `${BASE_URL}/api/v1/auth/phone_login`,
      method: 'POST',
      data: { phone: '13800138001' }
    });
    return response.data;
  }
  // ...
}
```

### 用户切换流程

```typescript
const handleSwitchUser = async (index) => {
  const user = TEST_USERS[index];
  
  // 1. 调用手机号登录API
  const response = await post('/api/v1/auth/phone_login', {
    phone: user.phone
  });
  
  // 2. 更新token
  setTokens(response.token, response.refreshToken);
  
  // 3. 刷新页面
  Taro.reLaunch({ url: '/pages/profile/index' });
}
```

## 🚨 注意事项

### 1. 仅开发环境可见

用户切换器只在开发环境下显示（`IS_DEV = true`），生产环境不会出现

### 2. 自动刷新

切换用户后会自动刷新整个小程序，确保所有数据都是新用户的

### 3. Token持久化

切换后的token会保存在localStorage，重启小程序也会保持登录状态

### 4. API权限

所有测试用户都有相同的权限（普通用户），可以查看自己的回忆录和广场上的公开回忆录

## 🔧 手动切换方法

如果需要手动切换，可以在微信开发者工具的Console中执行：

```javascript
// 方法1: 直接设置token
const response = await wx.request({
  url: 'http://localhost:8999/api/v1/auth/phone_login',
  method: 'POST',
  data: { phone: '13800138002' } // 李秀芳
});

wx.setStorageSync('token', response.data.data.token);
wx.setStorageSync('refreshToken', response.data.data.refreshToken);

// 刷新小程序
wx.reLaunch({ url: '/pages/home/index' });
```

```javascript
// 方法2: 使用curl获取token
// 在终端执行:
curl -X POST http://localhost:8999/api/v1/auth/phone_login \
  -H "Content-Type: application/json" \
  -d '{"phone": "13800138003"}'

// 复制返回的token，然后在Console中:
wx.setStorageSync('token', 'YOUR_TOKEN_HERE');
```

## 🎨 UI设计说明

### 颜色方案
- 开发模式面板：橙色系 (`amber`)
- 突出显示，与正常UI区分
- 清晰表明这是开发功能

### 交互反馈
- 切换时显示loading
- 成功后显示toast提示
- 自动刷新页面确保数据同步

### 用户体验
- 一键切换，无需手动输入
- 显示用户描述信息
- 操作流程简单明了

---

## 🔄 生产环境处理

在生产环境（`IS_DEV = false`）：
- 用户切换器**不显示**
- 使用真实的微信登录
- 用户只能看到自己的数据

---

## 💡 最佳实践

1. **测试前先切换用户**，确保查看正确用户的数据
2. **测试广场功能时**，可以随意使用任一账号（都能看到所有公开回忆录）
3. **测试私有功能时**，切换到对应用户查看其专属数据
4. **发现问题时**，记录当前使用的测试账号便于重现
